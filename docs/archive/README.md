# Archive README

The `docs/archive` directory preserves retired notes, experiments, or status snapshots that are no longer part of the active cleanup story. Use these guidelines when working with archives:

1. **Add a short intro** at the top of any new file before dropping it into this folder.
2. **Tag the date and author** inside the archived file so future readers know when and why it was shelved.
3. **Link back to the active docs** (for example, point to `docs/current-status.md`) so the context doesn’t get lost.

If you promote an archived document back into active status, move it up to `docs/` and update the new version with the current timestamp.
